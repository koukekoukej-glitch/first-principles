# First Principles

结构化深度分析 Skill。把复杂问题拆解为子问题，按类型匹配收敛/发散策略，系统收集证据后推导结论。每一步内建失败兜底机制——独立子 Agent 交叉验证拆解完备性，区分性推理对抗确认偏误。

适用于 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 及任何支持文件读写 + Shell 执行 + 子 Agent 调度的 LLM 框架。

---

## 特性

- **问题分类驱动策略** — 收敛型（答案存在于事实中）和发散型（答案需要被构造）使用不同的收集和推理策略，混合型问题逐个子问题匹配
- **双向信息收集** — 自底向上（从已有材料逐层扫描）+ 自顶向下（从领域通用知识逐层映射），两个方向交叉覆盖盲区
- **独立完备性校验** — 拆解完成后 spawn 干净上下文的子 Agent 独立推导"应该覆盖哪些方面"，与主分析对比找盲点。同一上下文内重推会被自己的拆解锚定，必须用独立注意力池
- **五个必要条件追踪** — 终态理解（NC1）、现状理解（NC2）、信息充分性（NC3）、推理可靠性（NC4）、传达有效性（NC5），每个条件有对应的保障步骤和失败兜底
- **两档架构按需切换** — 深潜（单 Agent 完整走完）适合大多数问题；围猎（多 Agent 并行收集 + 对抗审查）用于"漏一个就失败"级别的完备性需求

## 工作流程

```
步骤 1  理解与对齐 ── 确认终态和动机，对齐问题理解
│
步骤 2  拆解与分类 ── 按问题自然结构拆解，独立子 Agent 交叉验证完备性
│
步骤 3  系统收集   ── 每个子问题双向收集，收集完毕后缺口审计
│
步骤 4  推理与合成 ── 区分性推理，自我挑战，子解冲突检测
│
步骤 5  呈现       ── 结论先行逐层展开，未验证假设标注清楚
```

### 两种模式

| 模式 | 适用场景 | 验证机制 |
|------|---------|---------|
| **深潜** | 大多数有复杂度的问题：诊断 bug、设计架构、方案对比 | 单 Agent 完整走完，spawn 一次子 Agent 做拆解完备性对照 |
| **围猎** | "漏一个就失败"：安全审计、迁移评估、合规检查 | 侦察 Agent 绘制地形图，多收集 Agent 并行扫描，对抗 Agent 独立审查 |

深潜过程中发现完备性压力大可以升级到围猎，反过来不行。不确定时选深潜。

## 问题分类

所有问题归入两类或其混合。用错策略两边都会失败——枚举法做创造性子问题导致视野窄，发散法做诊断性子问题导致遗漏。

|  | 收敛型 | 发散型 |
|---|---|---|
| 本质 | 答案存在于事实中 | 答案需要被构造 |
| 瓶颈 | 信息完备性 | 生成质量 |
| 策略 | 系统枚举 → 排除 → 验证 | 约束识别 → 多候选生成 → 评估 |

## 失败模式防御

### 结构性防御

每个步骤的失败模式对自身不可见——误解问题时你不知道自己误解了，遗漏信息时你不知道自己遗漏了。所以每一步都有外部兜底：

| 必要条件 | 保障步骤 | 兜底机制 |
|---------|---------|---------|
| NC1 正确理解终态 | 理解与对齐 | 用户校验 |
| NC2 正确理解现状 | 理解与对齐 + 收集 | 用户校验 + 系统收集 |
| NC3 充分信息 | 拆解 + 收集 | 独立推导交叉 + 双向收集 + 缺口审计 |
| NC4 可靠推理 | 推理与合成 | 区分性推理 + 自我挑战 |
| NC5 有效传达 | 呈现 | 结论先行 + 用户反馈 |

### 纪律约束

- **没对齐就不分析** — 对抗"收到输入就产出答案"的倾向
- **能查到的不问用户** — 穷尽代码库、git 历史、网络搜索后才求助
- **结论强度不超过证据** — 没有证据标注为假设，不伪装成结论
- **"找到一些"不等于"找到全部"** — 完备性靠枚举框架和独立推导交叉，不靠主观感觉

## 围猎模式 Agent 架构

```
编排者（你）
├── 侦察 Agent     绘制问题域地形图，发现拆解前的信息盲区
├── 收集 Agent ×N  按维度并行收集，自底向上 + 自顶向下
├── 分析 Agent     合并发现，推理合成，产出初步结论
├── 挑战 Agent     对抗性审查：找漏洞、反例、未验证假设
└── 缺口扫描 Agent 最终缺口审计，确认每个维度都被覆盖
```

每个子 Agent 拥有独立的注意力池，编排者通过文件系统协调信息传递，不在自己上下文中持有完整产出。

## 设计原则

1. **策略跟随问题类型** — 收敛型和发散型用不同方法，不是所有问题都用同一套流程
2. **失败模式对自身不可见** — 每步都需要外部兜底，不信任"走完一步后看起来对了"的感觉
3. **完备性来自独立视角交叉** — 同一上下文内重推会被自己锚定，必须用独立注意力池
4. **延迟加载** — 阶段分离、分步读取，避免指令竞争注意力
5. **结论强度锚定证据** — 确认的坚定，不确定的具体说清缺什么
